### 明确任务边界

与其说"帮我修复这个bug"，不如具体说明：
- 期望的行为是什么
- 当前出了什么问题
- 相关的文件或代码位置

示例：
```
修复 note/devel/ai/claude-code.md 中的图片链接失效问题，第45行开始
```

### 利用上下文

Claude Code 能读取项目文件，直接问"这个模块的作用是什么"比复制粘贴代码更高效。

常用上下文获取方式：
```
/read <文件路径>    # 让AI读取文件内容
/refactor          # 在代码审查后重构
```

### 分步骤操作

复杂任务不要一次性交给 AI，分步骤可以：
1. 减少上下文膨胀
2. 及时发现偏差
3. 更好地控制风险

```
1. "先帮我分析这个函数的逻辑"
2. "基于以上分析，列出需要修改的地方"
3. "按顺序执行修改"
```

### 巧用 slash commands

| 命令 | 用途 |
|------|------|
| `/help` | 查看所有可用命令 |
| `/compact` | 压缩上下文节省 token |
| `/model` | 切换模型（Haiku/Sonnet/Opus） |
| `/browse` | 开启网页浏览模式 |
| `/skills` | 查看可用技能 |

### 内存功能

Claude Code 有持久化内存系统，可以在会话间记住：
- 用户偏好
- 项目上下文
- 工作流程约定

使用 `/remember` 或让 AI 自动记忆关键信息。

### 安全操作

对于敏感操作：
- 删除文件前先确认路径
- destructive 操作会让 AI 主动确认
- 使用 `/allow` 或 `/deny` 控制工具权限

### 好的 prompt 实践

**推荐：**
```
在 note/ 目录下创建一个新的博客文章模板
要求：
- 包含 title、publishDate、description、tags 四个字段
- 保存到 blog/devel/ 目录
```

**不推荐：**
```
帮我创建博客模板
```

越具体的指令，输出越符合预期。
